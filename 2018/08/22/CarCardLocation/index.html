写一个方便自己加油的APP

Posted on 2018-08-22 | In iOS Develop |

起因

今年买了个代步车，感觉出行方便多了。但是唰唰唰的几下，油用完了，唰唰唰的几下，油又用完了。于是乎，办了张招行的 carcard 加油打折卡（平时加油返3%，周五5%），某一天，兴高采烈地就近去加了个油。自己脑抽，没有问支不支持，就直接加了，完了告诉我 不支持 刷卡、微信、支付宝，仅现金。完犊子，新人不了解中石化的规矩，老老实实的去套现付了先。

回去一查原来要到固定的加油站加才行，网站上列了一堆地址，这让我个宅男如何反编码到地图上，一个个查，想想都累。反正周末不加班没啥事，写个简单的地图给自己导航一下好了。

思路与实现

先上个代码 https://github.com/fcgeek/CarCardLocation

  1. 获取城市列表，筛选出加油站列表。PS: 这里拿到加油站网页源码后可以选择引入三方库解析 HTML，或是 JavaScript 解析，前面分享过一篇 JavaScript 的用法，方便简洁又不需要引入其他库，相当好用；
  2. 使用 CLGeocoder 解析出地址的经纬度，保存 Realm 数据库方便下次直接使用，然后地图上标注出来；
  3. 点击加油站图标弹出导航；

只需三步，打完收工。

当前存在的问题

  1. CLGeocoder 每次到了一个地址就获取失败了，困扰了我很久，以为是网络或什么问题，最后确定是限制了一次编码的个数（50个，都是血泪得出的精确数字..），我现在是只能重进3-4次才加载好一个城市的加油站，希望后面有什么好的解决办法。想要一次完成很多个地址编码看来只能使用高德地图、百度地图这些地图的 API 了，但是看了一下他们的限制次数，还是算了。如果弄一个后端的话倒是方便，抓取之后存到后端，APP访问后端获取经纬度，这样就只需要编码一次，API 次数是完全没问题的。后面如果用着烦倒是可以考虑，不过似乎没必要了，重进几次以后就不需要了，后面如果项目有用到这种情况，倒是可以拿来用用；
  2. 自己的位置刷新自动跳回，不确定要不要去除，先留着；
  3. 测试跳转
